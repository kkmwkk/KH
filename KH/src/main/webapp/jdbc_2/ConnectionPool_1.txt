. ConnectionPool(커넥션풀)

: DBCP(DataBase ConnectionPool)

데이터베이스와 연결하는 커넥션을 미리 만들어서 pool 속에 저장해 두고 있다가 필요할 때 커넥션을 풀에서 쓰고 다시 풀에 반환하는 기법을 뜻한다.
웹 프로그램에서는 데이터베이스 환경설정과 연결 관리등을 따로 XML 파일이나 속성 파일을 사용해서 관리하고, 이렇게 설정된 정보의 이름을 사용하여 획득하는 방법을 말한다.

.커넥션 풀 사용 이유
만일 한명의 접속자가 웹 사이트에 접속할때 해당 웹 사이트에서 게시판을 확인하고 자신이 쓴 글을 수정하고 새로운 글을 등록한다고 가정하면
1) 데이터 취득
2) 검색후 데이터 취득
3) 데이터 갱신
4) 데이터 새로이 등록

와 같은 4번의 DB 접속이 일어난다
이럴때 많은 사람이 접속한다면 아주 많은 커넥션이 발생하게 된다.
커넥션풀은 미리 커넥션 객체를 만들어 놓고 해당 커넥션을 관리하는 것을 의미한다.

커넥션 풀의 특징 

풀속에 미리 커넥션이 생성되어 있기 떄문에 커넥션을 생성하는데 드는 연결시간이 소비되지 않는다.
커넥션을 계속해서 재사용하기 떄문에 생성되는 커넥션의 수가 많지 않다.
